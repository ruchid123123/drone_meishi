# 陀螺仪算法优化方案 (2026-04-21)

针对本项目（Drone Meishi）使用的 MPU6050 硬件及 Mahony/Madgwick 融合算法，提出以下优化路径，旨在提升飞行稳定性、减少姿态漂移并增强抗震动能力。

## 1. 硬件层优化 (MPU6050 配置)
*   **启用内置数字低通滤波器 (DLPF)**：目前驱动层可以增加对 `CONFIG (0x1A)` 寄存器的配置。建议将 DLPF 设置为 42Hz 或 20Hz，以在源头上滤除电机高频机械振动。
*   **采样率同步**：确保 MPU6050 的内部采样率（Sample Rate）与飞控主循环（500Hz）保持倍数同步，减少混叠噪声。

## 2. 核心算法改进 (Mahony/Madgwick)
*   **动态增益控制 (Adaptive Gains)**：
    *   **运动感知**：在检测到剧烈加速或旋转时，动态降低加速度计的反馈权重（Kp/Beta），防止非重力加速度干扰姿态。
    *   **静止校准**：在检测到长期静止时，提高权重以快速修正陀螺仪的累积漂移。
*   **加速度计阈值剔除 (G-vector Clipping)**：在 Mahony 算法中加入逻辑：当 `|Acc_total - 1G| > Threshold` 时，暂时停止加速度计的反馈修正。

## 3. 信号处理优化
*   **软件陷波滤波器 (Notch Filter)**：针对电机转速产生的特定频率干扰（通常在 150Hz-300Hz 之间），在 IMU 原始数据进入算法前增加陷波滤波，这比通用的低通滤波更精准且延迟更低。
*   **零偏动态温度补偿**：利用 MPU6050 内置温度计，建立简单的线性模型 `Bias = k * Temp + b`，解决无人机工作一段时间发热后产生的侧偏现象。

## 4. 系统执行优化 (ESP32 特性)
*   **多核并行处理**：将 `IMU Read + AHRS Update` 任务锁定在 ESP32 的 Core 0 运行，而将 `Web UI + WiFi` 锁定在 Core 1。避免网络协议栈阻塞导致的姿态计算抖动。
*   **使用硬件浮点优化**：确保算法实现中大量使用 `float` 而非 `double`，并调用 ESP32 的 `rsqrtf` 等 FPU 优化函数进行四元数归一化。

## 5. 预期效果
*   **抗干扰性**：在电机高速转动下，姿态抖动幅度预计降低 30% 以上。
*   **响应速度**：通过减少软件滤波延迟，提升飞控对外部扰动的纠正速度。
*   **长时稳定性**：航向漂移和水平漂移在 5 分钟飞行任务中将更加线性可控。
